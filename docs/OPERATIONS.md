# FOUDY Operations Manual

## 1. Deployment
FOUDY is configured for PaaS deployment.
- **Frontend:** Deployed to Vercel (via `vercel.json`). Pushes to `main` trigger auto-deployments.
- **Backend:** Deployed to Render (via `render.yaml`). Includes a Web service (`daphne` for ASGI) and a Background Worker (`celery`).

## 2. Environment Variables Guide
### Backend (`backend/.env`)
- `SECRET_KEY`: Django cryptographic key. (Auto-generated by Render in production).
- `DATABASE_URL`: Connection string to Neon PostgreSQL.
- `REDIS_URL`: Connection string to Upstash Redis (used for Cache, Channels, Celery).
- `RESEND_API_KEY`: Mail delivery provider.
- `CLOUDINARY_URL`: Storage provider for media.
- `TURN_URL` & `TURN_SECRET`: Metered TURN server for WebRTC relay (e.g., Twilio/Metered).

### Frontend (`frontend/.env`)
- `VITE_API_URL`: URL of the backend API.
- `VITE_WS_URL`: URL of the backend WebSocket.

## 3. Incident Response
1. **Downtime / 502 Bad Gateway:** 
   - Check Render logs for the web service. Ensure database and Redis are reachable.
   - Run health checks: `/health/system/`, `/health/database/`, `/health/redis/`.
2. **WebRTC Connection Failures:**
   - Verify `TURN_URL` credentials are valid. If STUN works but TURN fails, users on symmetric NATs will not connect.
3. **Task Queue Backup:**
   - Check Celery worker logs. Ensure Redis memory is not full.

## 4. Backups and Disaster Recovery
- **PostgreSQL:** Neon provides automated point-in-time recovery (PITR) and branching. Configure weekly logical dumps via `pg_dump` to an S3 bucket for cold storage.
- **Redis:** Upstash Redis is configured with multi-AZ replication. No manual backups required for ephemeral data.
- **Media:** Cloudinary retains all uploaded media. Ensure Cloudinary backup rules are configured in the Cloudinary console.
